# CONTRIBUTING

このリポジトリは有志のコミットで改善されていくことを歓迎する。

## スタンス

- 一人の正解より、現場の知見の積み重ねを重視する
- IT知識が浅い人にも届く言葉で書くことを心がける
- 完璧でなくてもよい。叩き台でも歓迎する

## 歓迎する貢献

- 業務ツリー（@README.md）への分類追加・修正の提案
- 各業務領域の詳細ページ（あるべき姿・ベストプラクティス）の執筆・加筆
- 誤字脱字、表現の改善
- 参考資料・出典の追加

## 進め方

- 小さい修正は直接 Pull Request を歓迎する
- 大きな構造変更（ツリーの再編、フォルダ構成の変更など）は Issue で相談してから進めること
- 表記・口調・命名規則などのルールは @AGENTS.md を参照
