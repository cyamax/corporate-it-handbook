# 全社DX・変革の推進

## 概要

デジタルを手段に、事業モデル・顧客価値・組織・文化そのものを変えていく全社的な変革を、横断でドライブする最上位の推進機能。個別のシステムやサービスを作る業務ではなく、**会社全体の変革の方向を定め、旗を振り、人と組織を動かし、機運をつくる**ことが主眼である。経営アジェンダであり、社内ITはその推進エンジンの一翼を担う。

> 枝＝変革 > 全社DX・変革の推進 ／ フェーズ軸＝発見・相談 → 現状分析・要件 → 企画・提案 → 構築・導入 → 定着化 → 効果測定・改善

> DX（デジタルによる事業・組織の変革）は「IT化・デジタル化」とは別物である。ツールを入れることが目的ではなく、デジタルを手段に事業や顧客価値・働き方を変えることが目的である。手段の導入で満足する「なんちゃってDX」を本ノードは戒める。

---

## あるべき姿

- **目的が事業・顧客にある** … DXが「ツール導入」「システム刷新」そのものを目的にしておらず、事業モデル・顧客価値・組織の変化という到達点に向いている
- **経営が当事者** … DXが情シス・DX部門の旗振りに閉じず、経営が自らの課題として旗を持ち、意思決定とリソース配分にコミットしている
- **全社が巻き込まれている** … 一部の推進部門だけが旗を振るのではなく、事業部門・現場が変革の主体として参画している
- **文化・人が変わっている** … 道具だけでなく、働き方・意思決定の仕方・学ぶ姿勢といった組織文化と人のスキルが変わりつつある
- **小さな成功が積み上がる** … 大号令だけで終わらせず、クイックウィン（小さく早い成功体験）で手応えと推進力を生んでいる
- **成果が数字で見える** … 変革の進み具合と成果がKPI（重要指標）で見え、続ける・広げる・見直すの判断ができている
- **推進に徹している** … 個別の実装を抱え込まず、方向づけ・調整・支援に徹し、実装は各変革ノードや事業部門が担う役割分担ができている

## 基本方針（汎用）

会社や事業が違っても、全社DXの推進で外せない土台は共通する。

- **DXはIT化ではない（手段と目的を分ける）** … デジタルは手段であり、目的は事業・顧客価値・組織の変化である。ツールを入れた・システムを新しくした、で終わらせない。手段の導入を成果と取り違えないことが、本ノード最大の前提である
- **経営のコミットを引き出す** … 全社変革は情シス・DX部門だけが旗を振っても現場は動かない。誰が責任を持ち、どこに投資するかを経営が決める状態を作る。推進部門の仕事は、経営が判断できる材料と機運を用意することでもある
- **巻き込みが主戦場である** … 変革の難所は技術ではなく組織である。抵抗・既得権・部門間の壁（サイロ）と向き合い、現場を変革の主体に変えていく。号令ではなく対話と実利で動かす
- **文化・人・スキルを変える** … 道具の導入と並行して、働き方・意思決定の仕方・学ぶ姿勢を変え、デジタルを使いこなす人材とリテラシー（基礎的な活用力）を育てる。組織と人が変わらなければ変革は定着しない
- **クイックウィンで機運を作る** … 全社変革は成果が見えにくく、機運が続かない。小さく早く出せる成功体験を意図的に作り、見せ、横へ広げて推進力に変える
- **推進・調整・支援に徹する** … 個別の実装を推進部門が抱え込むと、旗振りが止まる。実装は各変革ノードや事業部門へ渡し、本ノードは方向づけ・全社調整・支援に集中する
- **戦略の策定・投資管理は統制に返す** … IT戦略そのものの策定や投資ポートフォリオの管理は別の役割である。本ノードは「決まった変革の方向を全社で前へ進める」推進機能であり、戦略立案・投資配分は統制と連携して進める

## 主な業務

### 変革ビジョンと全社方針の旗振り

会社がデジタルで何を目指すのか、**到達したい姿と変革の物語を言語化し、全社へ繰り返し示す**。

- 「何のために変わるのか」を事業・顧客の言葉で語る。技術やツールの言葉だけで語らない
- 経営のメッセージとして発信できる形に整え、経営自身が旗を持って語る状態を作る。推進部門の代弁で終わらせない
- ビジョンを一度きりの号令で終わらせず、節目ごとに語り直し、現在地と次の一歩を示し続ける
- 個別施策がビジョンと結びついて見えるようにし、現場が「自分の仕事がどこに効くか」を理解できるようにする

### 全社横断の推進体制と巻き込み

部門の壁を越えて変革を進めるための、**体制・対話・合意形成の場をつくり、現場を主体にしていく**。

- 推進の責任者・推進部門・事業部門の役割を定め、誰が何を決めるかを明確にする
- 部門間の利害調整やサイロの解消を担い、全社最適の視点で交通整理をする
- 現場のキーパーソンを巻き込み、変革の主体（推進役・アンバサダー）を各部門に増やす
- 抵抗・不安には対話で向き合う。一方的な押し付けではなく、現場の困りごとを起点に変革の意義を翻訳する

### 変革文化の醸成と人材・リテラシー育成

道具より難しい、**人と組織の変化**を支える。学ぶ姿勢・試す姿勢・データで考える姿勢を全社へ広げる。

- 全社員のデジタルリテラシー（基礎的な活用力）を底上げする教育・研修を企画する
- 変革を担う人材（推進人材・デジタル人材）の育成・確保を、統制（IT組織・人材マネジメント）と連携して進める
- 小さく試すこと・失敗から学ぶことを許容する文化を醸成し、減点主義で挑戦が止まらないようにする
- 成功事例・ノウハウを全社で共有し、横展開できる土壌を作る

### 成果の可視化とクイックウィンの創出・定着

変革の進み具合と効果を見える化し、**小さな成功を作り・見せ・広げて**、変革を続く力に変える。

- 全社のDX推進度・成果をKPIで可視化し、経営と現場が同じ数字で進捗を語れるようにする
- 早く成果が出る領域を選んでクイックウィンを作り、成功体験を全社へ見せて機運を高める
- うまくいった取り組みを横展開し、点の成功を面の変化に広げる
- 定着しない取り組みは原因を見極め、続ける・広げる・見直すを判断する

## フェーズ別の進め方

変革のフェーズ軸（発見・相談 → 現状分析・要件 → 企画・提案 → 構築・導入 → 定着化 → 効果測定・改善）に沿って示す。全社推進という性質上、各段は個別案件の進行ではなく、**全社変革のうねりを起こし続けるサイクル**として読む。

| フェーズ | やること | 成果物 | つまずきやすい点 |
|---|---|---|---|
| 発見・相談 | 経営・事業の課題と危機感を引き出し、変革の必要性を共有する | 課題仮説・経営との問題意識の共有 | 情シス起点の独り相撲になる／危機感が経営・現場に共有されない |
| 現状分析・要件 | 全社の現状（業務・組織・文化・デジタル成熟度）を把握し、変えるべき本質を見極める | 現状評価・変革テーマ・優先領域 | デジタル化の遅れだけを見て、組織・文化の課題を見落とす |
| 企画・提案 | 変革ビジョン・推進体制・ロードマップ・KPIを描き、経営の合意を得る | 変革ビジョン・推進計画・KPI設計 | ツール導入計画にすり替わる／経営の本気のコミットを取れないまま走り出す |
| 構築・導入 | 推進体制を立ち上げ、クイックウィンを作りながら全社へ巻き込みを広げる | 推進体制・初期の成功事例・教育プログラム | 大上段の構想だけで現場が動かない／小さな成功を作らず機運が萎む |
| 定着化 | 成功を横展開し、文化・人・働き方の変化として根づかせる | 横展開された施策・育成された人材・定着した行動 | 道具は入ったが使われ方・文化が変わらない／推進部門が抜けると元に戻る |
| 効果測定・改善 | 事業・顧客への効果をKPIで測り、続ける・広げる・見直すを判断する | 成果レポート・次の変革テーマ・見直し判断 | ツール導入数など手段の指標だけ見て、事業価値の変化を測れない |

## やらないこと

全社DXの推進は「変革に関わること全部」を背負い込みやすい。**あえてやらない・抱え込まないこと**を明確にし、旗振り・全社推進という本来の役割を守る。

- **個別の効率化を抱えること** … 既存業務のコスト削減・省力化が目的の取り組みは、手段がツール・自動化・内製であっても変革ではない。`改善 > 業務プロセス改善・標準化` や `改善 > 手作業の自動化` など改善側へ振り分ける。目的が効率化なら改善、新価値創出なら変革という境界を守る
- **個別の実装を抱え込むこと** … 新サービス・データ活用・先進技術・システム刷新の実装は、それぞれ `変革 > 新規事業・新サービスのIT支援` `変革 > データ活用による新価値創出` `変革 > 先進技術による競争力強化` `変革 > 既存システムの刷新・モダナイゼーション` が主管する。本ノードは旗振り・全社推進であり、個別案件の実装を抱え込まない
- **IT戦略の策定そのものを担うこと** … 中長期のIT戦略・ロードマップの策定は `統制 > IT戦略・企画` が担う。本ノードは決まった変革の方向を全社で前へ進める推進機能であり、戦略立案そのものとは役割が違う
- **投資ポートフォリオの管理を担うこと** … 案件の優先順位づけ・予算配分・全社横断の進捗管理は `統制 > プロジェクト・ポートフォリオ管理（PMO）` が担う。本ノードは投資配分の意思決定機関ではない
- **DXをデジタル化・IT化に矮小化すること** … ツールを入れる・システムを新しくすることをDXの成果と取り違えない。手段の導入を目的化した「なんちゃってDX」を推進しない。常に事業・顧客・組織の変化に立ち返る
- **経営の意思決定を肩代わりすること** … どこへ投資し何を変えるかの最終判断は経営が握る。推進部門が経営の代わりに決めると、当事者意識が育たず、変革が「情シスの取り組み」に矮小化される

> ここで線を引くのは消極姿勢ではない。**推進部門が旗振り・調整・支援に集中し、実装を各ノードへ、戦略・投資管理を統制へ、最終判断を経営へ返す**ことで、全社変革のうねりを止めないためのものである。

## よくある課題

- **ツール導入で満足する** … グループウェアやSaaSを入れたことを「DXした」と捉え、事業も組織も変わらないまま終わる（なんちゃってDX）
- **情シス・DX部門の独り相撲** … 推進部門だけが旗を振り、経営も現場も他人事のまま、号令だけが空回りする
- **経営のコミットがない** … 経営が口では推進と言うが、リソースも意思決定も伴わず、現場が「どうせ本気ではない」と動かない
- **現場の抵抗・サイロ** … 既得権や部門の壁にぶつかり、全社最適の取り組みが各部門の都合で骨抜きになる
- **文化・人が変わらない** … 道具は入ったが、意思決定の仕方も学ぶ姿勢も旧来のままで、変革が定着しない
- **成果が見えず機運が続かない** … 全社変革は効果が見えにくく、小さな成功も示せず、いつの間にか熱が冷める
- **推進部門が実装を抱え込む** … 旗振り役が個別案件まで丸抱えし、手が回らず、肝心の全社推進が止まる

## ベストプラクティス

- **経営をオーナーに据える** … DXの責任者を経営層に置き、経営自身が旗を持って語る状態を作る。推進部門は経営が判断・発信できる材料を用意する役に回る
- **危機感と目的を事業の言葉で語る** … 「なぜ変わるのか」を技術ではなく事業・顧客の言葉で語り、変わらないことのリスクを共有する。共感が巻き込みの起点になる
- **クイックウィンを設計する** … 早く成果が出る領域を意図的に選び、小さな成功を作って見せる。成功体験を横展開の燃料にする
- **現場に推進役を増やす** … 各部門にアンバサダー・推進役を置き、現場の言葉で変革を翻訳してもらう。中央集権の押し付けにしない
- **リテラシー教育を全社で回す** … 全社員の基礎的なデジタル活用力を底上げし、一部の人だけが使える状態をなくす。育成は統制（IT組織・人材マネジメント）と連携する
- **失敗を許容する文化を作る** … 小さく試し、失敗から学ぶことを評価する。減点主義で挑戦が止まる空気を変える
- **DX推進度をKPIで可視化する** … 手段の指標（導入数）ではなく、事業・顧客・働き方の変化に効く指標で進捗を測り、経営と現場が同じ数字で語れるようにする
- **役割分担を明文化する** … 推進部門・各変革ノード・統制・事業部門の担当範囲を文書で線引きし、抱え込みと取りこぼしの両方を防ぐ

## 成熟度モデル

自社の現在地を測る目安。下から積み上げる。

| Lv | 状態 | 特徴 |
|---|---|---|
| Lv1 デジタル化止まり | 手段が目的 | ツール導入・システム化を「DX」と呼んでいる。事業も組織も変わらず、情シス主導の局所的な取り組みに留まる |
| Lv2 推進体制あり | 旗は立った | DX推進の方針・体制ができ、経営が旗を持ち始めている。ただし現場は他人事で、巻き込みはこれから |
| Lv3 全社で動く | 巻き込みが回る | 事業部門が変革の主体になり、クイックウィンが横展開され、文化・人の変化が始まっている |
| Lv4 変革が常態 | 事業と一体 | 事業モデル・顧客価値の変化が数字で見え、変わり続けることが組織の当たり前になっている。デジタルが経営に織り込まれている |

> 段階は目安であり、現実には重なって進む。ただし**まず Lv2（経営が旗を持つ推進体制）を固める**こと。経営のコミットがないまま推進部門だけが走っても、Lv3 以降の全社の巻き込みは成り立たない。
