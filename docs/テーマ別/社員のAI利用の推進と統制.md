# 社員のAI利用の推進と統制（読み手：情シス）

## 概要

社員が業務でAI（人工知能、ここでは主に生成AI＝文章・画像・コード等を作り出すAI）を使う場面が急速に広がっている。このページは、社員本人向けの利用ガイドそのものではなく、情シスが**全社員のAI利用をどう安全に推進し、ガードレール（はみ出しを防ぐ仕組み）をどう設計・運用するか**を扱う。推進と統制は対立しない。安全に使える状態を作ることが、結果的に最大の統制になる。

> 観点＝社員のAI利用の推進と統制 ／ 読み手＝情シス（社員本人ではなく、社員のAI利用を支援・統制する立場。IT知識が浅いメンバーも含む）

このテーマは特定の業務ノードに収まらない。利用ルールは統制、日々の問い合わせ対応とシャドーAI対応は運用、定型業務の効率化支援は改善、新価値創出への活用は変革にまたがる。そのため横断テーマとして扱う。

---

## 基本スタンス

AI利用に臨むうえで、情シスが土台に置くべき考え方を示す。

- **「使わせない」ではなく「安全に使える状態を作る」** … 禁止一辺倒は、社員が個人アカウントでこっそり使うシャドーAI（情シスの管理外で無断利用されるAI）を生むだけである。管理外で使われれば、情報漏洩も誤用も情シスからは見えなくなる。塞ぐより、安全な道を用意して誘導するほうが統制が効く
- **推進と統制は両輪** … 認める道（推奨ツール・事例の共有）と、守る線（入れてよい情報の区分・教育）を同時に整える。片方だけでは機能しない
- **目的は効率化と新価値の両方** … 既存業務を楽にする使い方（効率化）と、新しい価値を生む使い方の両方を視野に入れる。どちらかが上ということはなく、どちらも安全に支える対象である
- **出力の責任を社員任せにしない** … AIはハルシネーション（もっともらしく見える誤り）を出す。出力をそのまま使ってよいか、誰が確認するかという会社としての方針を情シス側から示す。「使った社員の自己責任」で放置しない
- **ルールはわかりやすく、現場で守れる形に** … 厳密でも読まれず守られないルールは、無いに等しい。やってよい例・ダメな例を具体的に示し、現場が判断に迷わない粒度にする

## 業務区分ごとの観点

社員のAI利用は、運用・改善・変革・統制の4区分すべてに関わる。区分ごとに情シスの関わり方を示す。重なる本体は各ノードへ参照する。

### 統制：ルールと方針を定める

AI利用の土台となる方針・規程・データの線引きを決める区分。横断テーマであるこのページの中心であり、本体は統制ノード側にある。

- **利用ポリシー・規程の本体** … 何に使ってよい・ダメか、利用時の遵守事項、責任の所在を定める。AI利用ポリシーは全社ITポリシーの一部として `統制 > ガバナンス・統制` で整備する。著作権・個人情報・法令遵守の論点もここに含む
- **入れてよい情報／入れてはいけない情報の線引き** … 機密情報・個人情報・顧客データをAIに入力してよいかの判断基準を定める。この線引きは全社のデータ区分（公開・社内限り・機密など）を土台にするため、本体は `統制 > データガバナンス` に従う。AI固有の追加ルールだけをこのテーマで足す
- **教育・リテラシーの位置づけ** … AI教育は単発ではなく、全社ITリテラシー教育の一部として継続的に回す。本体は `統制 > IT組織・人材マネジメント`

### 運用：日々の問い合わせとツール提供を回す

定めた方針を、日々の現場で回す区分。

- **問い合わせ対応** … 「このデータを入れてよいか」「どのツールを使えばよいか」という社員の質問に答える窓口を持つ。一般的なヘルプデスク運用は `運用 > ユーザーサポート`
- **認可ツールの提供運用** … 会社として認めるAIツールを配布・管理し、アカウント・利用枠を整える
- **シャドーAIの検知と是正** … 管理外のAI利用を見つけ、正規ルートへ誘導する。検知の仕組み、利用ログ・監査、情報漏洩への対応といった運用面は `運用 > セキュリティ運用` に従う

### 改善：定型業務の効率化を後押しする

社員がAIで既存業務を楽にする動きを支える区分。

- 議事録の要約、文章の下書き、定型メールの作成、表計算の補助といった、効率化が目的の使い方を後押しする
- 良いプロンプト（AIへの指示文）や成功事例を集めて横展開し、一部の社員の工夫を全社の標準にする
- 大がかりな業務効率化は `改善 > 手作業の自動化（RPA・スクリプト）` や `改善 > 業務プロセス改善・標準化` と地続きになる

### 変革：新価値創出への活用と接続する

AIを新しい価値づくりに使う動きとつなぐ区分。

- 効率化にとどまらず、新サービスや顧客体験に生成AIを組み込む取り組みは、新価値創出が目的のため `変革 > 先進技術による競争力強化` の領域に入る
- 社員の日常的なAI利用（このテーマの主対象）と、事業としてのAI活用（変革ノード）は地続きであり、現場で生まれた使い方が新価値の種になることもある。情シスは両者を分断せず橋渡しする

## やらないこと・落とし穴

AI利用の推進と統制で、情シスが陥りやすい失敗を挙げる。いずれも「禁止しすぎ」と「野放し」の両極に偏ったときに起きる。

- **全面禁止でシャドーAIを誘発する** … 「リスクがあるから一律禁止」は最も危うい。社員は個人アカウントで隠れて使い始め、利用実態が情シスから完全に見えなくなる。禁止は統制ではなく、統制の放棄になりやすい
- **ルールだけ作って教育・周知をしない** … 立派なポリシーを作っても、読まれず守られなければ棚に眠るだけである。作って終わりにせず、周知と教育まで一続きで設計する
- **技術的に塞ぐだけで使い方を示さない** … アクセスをブロックするだけでは、社員は別の手段を探すか、AIを使う改善機会を丸ごと失う。塞ぐと同時に「ではどう使えばよいか」の道を示す
- **社員の自由な工夫を過剰に縛る** … 細かすぎる承認や禁止項目は、現場の創意工夫を殺し、結局守られない。守るべき一線（機密を入れない等）に絞り、それ以外は委ねる
- **逆に野放しで情報漏洩を招く** … 線引きも教育もないまま自由に使わせれば、機密・個人情報・顧客データが管理外のAIに流れ込む。「推進」を「放任」と取り違えない
- **出力責任を社員任せにする** … ハルシネーションの確認も、著作権の判断も「使った人の責任」で済ませると、会社として方針が無いことの言い換えになる。確認の手順と責任の所在を会社として示す

> 統制とは「禁止」ではなく「安全に使える状態へ導くこと」である。線を引く目的は、社員の手を縛ることではなく、安心して使える範囲を明示することにある。

## 進め方・ベストプラクティス

段階的に整える。最初から完璧を目指さず、まず線引きと認可ツールという土台を固める。

1. **方針とデータの線引きを決める** … AIを何に使ってよいか、どの情報を入れてよい・ダメかを先に決める。本体ルールは `統制 > ガバナンス・統制`、データ区分は `統制 > データガバナンス` に乗せる
2. **認可ツールを用意する** … 会社として使ってよいAIツールを選び、提供する。無料版・個人アカウント利用の扱い（業務データ入力の可否など）も明確にする。使える道が無ければ社員は管理外のツールに流れる
3. **わかりやすいルールと事例を周知する** … 社員向けには「やってよい例・ダメな例」を具体的に示す。抽象的な禁止条項より、現場が判断できる具体例が効く
4. **教育する** … 出力を鵜呑みにしない、機密情報を入力しない、著作権に配慮する、という最低限のリテラシーを継続的に伝える。一度きりの研修で終えない
5. **利用状況を見て見直す** … 認可ツールの利用ログやアンケートで実態を把握し、シャドーAIの兆候や新しいニーズを拾って、ルールと提供ツールを更新する

社員向けに配るルールに含めるべき要素は、最低限次のとおりである。

- **入れてよい情報・入れてはいけない情報** … 機密・個人情報・顧客データの扱いを、具体例つきで
- **使ってよいツール・使ってはいけないツール** … 認可ツールの一覧と、無料版や個人アカウントの可否
- **出力の扱い** … 鵜呑みにせず人が確認すること、最終成果物の責任は使い手にあること
- **著作権・権利への配慮** … 他者の権利を侵害しうる入力・出力への注意
- **困ったときの相談先** … 判断に迷ったら情シスに聞ける窓口

## 成熟度モデル

自社の現在地を測る目安。Lv1には「野放し」と「全面禁止」の二つの形があり、どちらも危険である点に注意する。

| Lv | 状態 | 特徴 |
|---|---|---|
| Lv1 野放し or 全面禁止 | 両極端（どちらも危険） | ルールがなく自由に使われている（情報漏洩の温床）か、一律禁止でシャドーAIが横行している。いずれも実態が見えない |
| Lv2 ルール策定 | 線引きを決めた | 利用ポリシーとデータの線引きを定めた。ただし周知・教育・認可ツールはこれから |
| Lv3 安全に推進 | 認可ツール＋教育 | 認可ツールを提供し、わかりやすいルールと事例を周知し、教育を回している。社員が安心して使える |
| Lv4 継続改善 | データで前へ | 利用状況をログ・アンケートで把握し、シャドーAIの兆候や新ニーズを拾って、ルールとツールを継続的に見直している |

> Lv1の二つの形は正反対に見えて、どちらも「実態が見えない」点で同じ危うさを持つ。まず Lv2（線引き）と Lv3（認可ツール＋教育）を固め、推進と統制を両立させることを優先する。
