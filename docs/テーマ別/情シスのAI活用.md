# 情シスのAI活用（読み手＝情シス／観点＝情シス自身の業務にAIをどう活かすか）

## 概要

情報システム部門（情シス）が、自分たちの日々の業務にAI（人工知能）をどう活かすかを扱う横断テーマ。問い合わせ対応の下書き、ログの解読、手順書やポリシーのたたき台づくりなど、情シスの仕事の多くは「言葉を読み書きする作業」であり、生成AI（文章・画像などを生成するAI）と相性がよい。

> 観点＝情シス自身のAI活用 ／ 読み手＝情シス（IT知識が浅いメンバーを含む）

ここで重要なのは、**AIは業務ツリーの新しい区分ではない**ということ。このハンドブックの大原則どおり、AIはあくまで「手段」であり、目的（効率化＝改善／新価値創出＝変革）によって所属する区分が決まる。AIを使うこと自体を独立した業務分類に立てると、手段と目的の境界が崩れる。そのため、AIは運用・改善・変革・統制の4区分すべてに横串で関わる手段として、この横断テーマで扱う。

---

## 基本スタンス

会社や使うツールが違っても、情シスがAIに向き合う土台となる考え方は共通する。

- **AIは判断を代替しない** … AIは人の判断を「速く・広く」する道具であって、判断そのものを肩代わりするものではない。最終的に決めるのも、その結果に責任を持つのも人である
- **出力は必ず人が検証する** … AIはハルシネーション（AIがもっともらしく出す誤り）を起こす。事実・手順・設定値・引用元は、採用する前に人が裏取りする。検証できない出力は使わない
- **機密の線引きを先に決める** … 入力した情報がどこへ送られ、学習に使われるかは、利用規約とデータの送信先を確認してから判断する。確認が済むまで、機密情報・個人情報をAIに入力しない
- **解く課題が先、道具は後** … 「AIで何かできないか」ではなく「この課題をどう解くか、その手段としてAIは有効か」の順で考える。道具から入ると、解く必要のない問題に労力を割く
- **情シスが手本になる** … 全社にAI活用を広げる立場として、情シス自身が機密の扱い・検証の習慣を先に守り、見本を示す

---

## 業務区分ごとの観点

AIは運用・改善・変革・統制の4区分すべてに関わる。区分ごとに「情シスが自分の業務でどう使うか」を示す。業務ノードと重なる中身はそのノードを参照し、ここでは重複して深掘りしない。

### 運用（Run）── 今あるものを止めない

日々の窓口対応や保守で、言葉を扱う作業の下書きにAIを使う。

- **問い合わせの一次回答ドラフト** … よくある問い合わせへの返信文をAIに下書きさせ、人が確認して送る。窓口運用そのものの考え方は `運用 > ユーザーサポート` に従う
- **ナレッジ・FAQ（よくある質問集）の生成** … 過去の対応記録から、FAQ記事や手順書の下書きを起こす。最終的な内容の正しさは人が確認する
- **ログ・エラーメッセージの解読補助** … 見慣れないエラー文やログの意味を、AIに説明させて当たりをつける。ただし示された原因や対処は鵜呑みにせず、一次情報（公式ドキュメント・設定）で裏取りする
- **手順書の下書き** … 作業手順を箇条書きで渡して文章化させ、人が実機で確認して仕上げる

### 改善（Grow）── 今あるものを良くする・安くする

効率化を目的とした作業に、AIを補助として使う。なお、効率化のための実装の主担当は改善区分の各ノードであり、ここでは情シス自身の作業効率化という観点で触れる。

- **定型作業の自動化スクリプト生成の補助** … 繰り返す手作業を自動化するスクリプトの下書きをAIに作らせる。動作確認とセキュリティ面の確認は人が行う。自動化の取り組み自体は `改善 > 手作業の自動化` に位置づく
- **議事録・報告書の要約** … 長い記録から要点を抽出し、要約の下書きを作る。要約が元の意図を歪めていないか人が照合する
- **業務プロセスの棚卸しの補助** … 手順や課題の洗い出しをAIに壁打ちさせ、改善の出発点にする。仕組みの標準化は `改善 > 業務プロセス改善・標準化` で扱う

### 変革（Transform）── 新しい価値を生む

新技術としてのAIで会社の競争力を生む取り組み自体は、`変革 > 先進技術による競争力強化` 側に位置づく。この横断テーマで扱うのは、**情シスが企画・検証を支援する**という観点である。

- **企画段階の壁打ち・調査の補助** … 事業側のAI活用案について、論点整理や前提調査の下書きをAIで進め、検討を速める
- **小さな検証（PoC：Proof of Concept、概念実証）の支援** … 実現性を見極めるための試作・評価を情シスが手伝う。新価値創出が目的のAI活用そのものの主担当は変革区分であり、ここはその支援に徹する

### 統制（Govern）── 横串で方向づけ・規律を与える

ポリシーやリスク評価など、文章量が多く下書きの効くものにAIを使う。ただし最終判断と責任は人にある。

- **ポリシー草案のたたき** … AI利用ガイドラインや規程の下書きをAIに作らせ、自社の実態に合わせて人が直す。規程・統制の実体は `統制 > ガバナンス・統制` に従う
- **リスク評価のたたき** … 想定されるリスクの洗い出しの下書きを作る。評価の妥当性と抜け漏れは人が確認する
- **ベンダー資料の要約のたたき** … 長い提案書・契約書の要約を作り、論点把握を速める。契約・選定の判断は `統制 > ベンダー・契約管理` に従い、最終確認は必ず原文で行う
- **AI利用そのもののルール整備** … 全社が安全にAIを使うための線引き・教育は統制の仕事として整える。組織的なリテラシー教育は `統制 > IT組織・人材マネジメント` に関わる

## やらないこと・落とし穴

便利さゆえに踏み外しやすい。**あえてやらないこと**と陥りやすい失敗を明確にする。

- **機密・個人情報を無確認で外部AIに入力すること** … 送信先と利用規約を確認しないまま、顧客情報・人事情報・社内機密・認証情報をAIに貼り付けない。一度送った情報は取り戻せない
- **出力を無検証で採用すること** … もっともらしい文章でも誤り（ハルシネーション）が混じる。手順・設定値・法令・引用元を裏取りせずに本番へ反映しない
- **AI導入自体を目的化すること** … 「AIを入れた」を成果にしない。解くべき課題が先にあり、AIはその手段にすぎない。課題が曖昧なまま導入すると、効果が測れず形骸化する
- **特定ツールへの過依存** … 一つのツール・サービスの作法に業務を縛りつけない。仕様変更・値上げ・提供終了で業務が止まるリスクを抱える。乗り換え可能な使い方を意識する
- **ベンダーの誇大宣伝の鵜呑み** … 「これだけで解決」「人手が不要になる」といった売り文句を、自社で検証せずに信じない。デモと実運用は別物である
- **生成物の著作権・ライセンスの無視** … AIの生成物の権利関係・利用条件・学習元の扱いを確認せずに、社外公開物や製品へ流用しない。後から権利問題が生じうる
- **AIに責任を負わせること** … 「AIがそう言ったから」は説明にならない。判断の責任は使った人と組織にある

> 線を引くことは「AIを使わない」ことではない。**安全に・効果的に使うために、入れてはいけない情報と、検証すべき出力を先に決めておく**ということである。

## 進め方・ベストプラクティス

- **小さく試す** … いきなり全社展開せず、影響の小さい範囲・機密でない作業から試す。失敗しても痛手にならない場所を選ぶ
- **効果と限界を見極める** … 試した結果、どこで効き、どこで外したかを記録する。「使える作業」と「任せてはいけない作業」を仕分ける
- **定着させる** … 効果が見えた使い方を手順・テンプレートに落とし、誰でも同じように使える形にする
- **プロンプトとナレッジを共有する** … うまくいった指示文（プロンプト）や注意点をチーム内で共有し、各自が一から試行錯誤する無駄をなくす
- **機密の線引きを明文化する** … 「何を入れてよく、何を入れてはいけないか」を文章で決め、情シスが先に守る。判断を個人の感覚に委ねない
- **検証を手順に組み込む** … 「AIに下書きさせる → 人が裏取りする → 採用する」を作業の標準フローにする。検証を省ける例外を作らない
- **使った跡を残す** … どの作業にAIを使ったかを記録に残し、後から品質や妥当性を振り返れるようにする

## 成熟度モデル

自社・自チームの現在地を測る目安。下から積み上げる。

| Lv | 状態 | 特徴 |
|---|---|---|
| Lv1 個人が各自で試す | 手探り | 興味のあるメンバーが個人の判断で使う。線引きもノウハウも共有されていない |
| Lv2 チームで使い所を共有 | 共有の始まり | 効く作業・避ける作業、機密の線引き、有効なプロンプトをチームで共有する |
| Lv3 業務フローに組み込み | 仕組み化 | 「下書き→検証→採用」が標準の作業手順に組み込まれ、誰が使っても一定の品質になる |
| Lv4 効果測定し継続改善 | データで前へ | 削減時間や品質を測り、使い方を見直し続ける。効かない使い方はやめ、効く使い方を広げる |

> 段階は目安であり、現実には重なって進む。ただし**Lv1の段階でこそ、機密の線引きと検証の習慣を先に固める**こと。土台がないまま広げると、情報漏洩や誤りの混入が組織全体に広がる。
