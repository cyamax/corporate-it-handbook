# オフィスIT・ファシリティ

## 概要

会議室のAV（音響・映像）機器、複合機、固定電話・IP電話（インターネット回線を使う電話）、執務席の配線・LAN（社内ネットワーク）など、**オフィスに物理的に置かれたIT設備**を、社員がいつでも当たり前に使える状態に保つ業務。地味だが「会議室で画面が映らない」「複合機が止まる」「電話が繋がらない」「席のLANが死ぬ」といった、物理的に人がその場で困る事象に直結する領域である。

> 枝＝`運用 > オフィスIT・ファシリティ` ／ フェーズ軸＝**受付 → 対応 → 解決 → 記録**（＋PDCA：Plan-Do-Check-Act、計画・実行・評価・改善のサイクル）

---

## あるべき姿

- **会議が時間どおり始まる** … 会議室に入って機材を繋げば、映像・音声・画面共有が迷わず動く。開始5分が機材トラブルで潰れない
- **印刷で困らない** … 複合機が動き、トナー・用紙が切れず、誰のジョブがどこから出るかが分かる
- **電話が繋がる** … 代表電話・内線・外線が止まらず、受けるべき相手に確実に繋がる
- **席に座れば仕事ができる** … どの席のLANポート・Wi-Fi（無線LAN）に繋いでも、すぐ社内ネットワークに入れる
- **誰の持ち物か分かる** … どの設備が「IT管理」で、どこからが「建物・総務の管理」かの境界が明確で、困ったときの行き先が決まっている
- **移転・レイアウト変更に耐える** … 席替え・増床・移転のたびに配線がぐちゃぐちゃにならず、図面と現物が一致している
- **止まっても止まりっぱなしにならない** … 予備機・予備在庫があり、壊れても短時間で代替に切り替えられる

## 基本方針（汎用）

会社や機種が違っても、オフィスIT設備を支える土台となる考え方は共通する。

- **物理は「その場で困る」前提で構える** … 会議室・複合機・電話・配線の障害は、リモートで直せないことが多く、人が現地に行くまで業務が止まる。**現地到達までの時間**そのものが品質になる
- **総務・ファシリティとの境界を先に引く** … 建物設備（電源工事・空調・内装・什器）は総務／ファシリティ、その上で動くIT機器はITが見る。境界を曖昧にすると、停電工事や席移動のたびに「どっちの仕事か」で止まる
- **設備を台帳で持つ** … 「どの会議室にどの機材があるか」「どの複合機がどこにあるか」「どの席のポートがどのスイッチに繋がるか」を台帳・図面で管理する。現物と図面の不一致が、移転時の事故の最大要因になる
- **予備を持つ** … 会議室のケーブル・アダプタ、複合機のトナー、電話機、LANケーブルは消耗・故障する前提で予備在庫を持つ。「1台壊れたら即アウト」の構成を避ける
- **標準化して機種を絞る** … 会議室ごとに違う機材・配線にしない。型を揃えるほど、操作も保守も予備も楽になる
- **使う人が迷わない導線にする** … IT担当者にしか使えない設備は失敗である。会議室には「繋ぐだけ」の手順、複合機には掲示、トラブル時の連絡先を現地に置く

## 主な業務

### 会議室AV（音響・映像）

会議室・応接・全社集会スペースの、**映す・聞こえる・繋がる**を担保する。

- ディスプレイ／プロジェクター、スピーカー・マイク、Web会議用カメラ、HDMI（映像・音声を1本で送るケーブル規格）等のケーブル類とアダプタを整備する
- 「PCを繋ぐだけで映る」状態を標準にする。会議室ごとに繋ぎ方が違う、特定のアダプタが無いと映らない、は事故のもと
- 予約システム（会議室予約）とディスプレイの連動、Web会議（リモート会議）端末の常設などは、利用頻度に応じて整える
- ケーブル・アダプタ・電池（リモコン・マイク）は消耗品として予備を常備する。会議直前に「映らない」が最も起きやすい

### 複合機・印刷環境

複合機（コピー・印刷・スキャン・FAXの一体機）と、社内の印刷の仕組み全体を見る。

- 設置・ネットワーク接続・ドライバ配布・印刷キュー（印刷の待ち行列）の管理を行う
- トナー・用紙の補充は総務と分担することが多い。**どちらが消耗品を持つか**を最初に決めておく
- 認証印刷（カードやIDで本人だけが出力できる仕組み）で、置き忘れによる情報漏洩を防ぐ
- 複合機はリース契約（ベンダーから借りる契約）が一般的。故障対応はベンダー保守が前提となり、ITは窓口と一次切り分けを担う
- 機器そのものの故障・保守はベンダー、印刷できない等の利用トラブルはITの一次対応、という分担を意識する

### 固定電話・IP電話

代表電話・内線・外線など、**会社の電話が繋がり続ける**ことを担保する。

- IP電話（インターネット回線を使う電話）・PBX（社内の電話交換機）・クラウドPBX（交換機機能をクラウドで提供する方式）の運用、内線番号・代表番号の管理を行う
- 入退社・席移動に伴う番号の付け替え、転送・着信ルールの設定を行う
- 回線契約・番号自体の管理はベンダー・通信事業者との連携が前提。**番号は会社の資産**であり、解約・移転時の番号維持（ナンバーポータビリティ）を取り違えると代表電話を失う
- スマートフォン・ビジネスチャットへの移行が進んでも、代表電話・受付・FAXなど「電話でないと困る」業務は残りやすい

### 配線・LAN（社内ネットワーク）

執務フロアの**物理的な繋がり**を担保する。配線盤からスイッチ、フロアのLANポート、各席までの経路を見る。

- フロアのLAN配線、パッチパネル（配線をまとめる接続盤）、フロアスイッチ（各席を束ねるネットワーク機器）、無線LANアクセスポイント（Wi-Fiの電波を出す機器）の設置・管理を行う
- 「どの席のポートが、どのスイッチの、どのポートに繋がっているか」の配線管理（結線情報）を台帳・ラベルで維持する
- 席替え・増床・レイアウト変更のたびに配線が発生する。**図面と現物を一致させ続ける**ことが、後の障害切り分けを左右する
- フロア内の物理配線・機器はITが見るが、その上位（基幹のネットワーク機器・回線）は `インフラ運用 > ネットワーク運用` の範囲。境界を意識する
- 床下・天井裏の配線敷設、電源コンセントの増設は、建物工事として総務／ファシリティ・電気工事業者の領域になる

## フェーズ別の進め方

運用のフェーズ軸（受付 → 対応 → 解決 → 記録）に沿って、各段でやること・つまずきを示す。

| フェーズ | やること | 成果物 | つまずきやすい点 |
|---|---|---|---|
| 受付 | 「映らない・印刷できない・繋がらない・LANが死んだ」を受け、設備・場所を特定する | 受付チケット（場所・設備の特定） | 現地に行かないと状況が分からない／総務案件かIT案件かの仕分けで止まる |
| 対応 | 現地で切り分け、予備機・予備部材で暫定復旧する。ベンダー保守の対象なら手配する | 対応ログ・暫定復旧・ベンダー手配 | 予備が無く復旧できない／会議中・営業時間中で作業できない |
| 解決 | 復旧を現地で確認し、利用者・会議主催者に通知してクローズする | 解決済みチケット | 「直ったつもり」で実機確認をせずクローズ／再発の前兆を見逃す |
| 記録（＋PDCA） | 対応内容・設備の状態を残し、消耗・故障の傾向を集計、予備在庫・標準機種・更新計画へ還元する | 設備台帳・配線図の更新・在庫レポート | 台帳・配線図を更新せず現物とずれる／消耗品切れが繰り返す |

## やらないこと

オフィスIT・ファシリティは「オフィスで物理的に何か困ったら、とりあえずIT」と見られやすい。**あえてやらない・抱え込まないこと**を明確にして、本来の役割と引き渡し先を守る。

- **建物設備・電気工事・内装をやること** … 電源コンセントの増設、分電盤・ブレーカー、床下／天井裏の配線敷設、空調、間仕切り・什器（机・椅子・パーティション）は **総務／ファシリティ・電気工事業者**の領域。ITは「その電源・配線の上で動くIT機器」を見るのであって、建物そのものは見ない
- **消耗品の在庫管理を全部抱えること** … 複合機のトナー・用紙、会議室の電池などの**発注・補充の主担当をどちらが持つか**を総務と決める。曖昧なまま全部ITが拾うと、本来の保守が消耗品の買い出しに埋もれる
- **端末そのものの管理** … PC・スマートフォン本体のキッティング・配布・回収・故障対応は `ユーザーサポート > 端末管理` の範囲。オフィスITが見るのは「席に座って繋ぐ先（LAN・Wi-Fi・会議室機材・複合機）」であり、端末そのものではない
- **社内ネットワークの上位** … 回線・基幹スイッチ・ルータ・全社のネットワーク設計は `インフラ運用 > ネットワーク運用` の範囲。オフィスITが見るのはフロアの物理配線・フロアスイッチ・アクセスポイントまでで、その上位は引き渡す
- **私物AV機器・私的利用の常時サポート** … 社員が持ち込んだ私物のスピーカー・モニター・変換アダプタや、業務外の私的利用のトラブルを恒常的に面倒見ない。会社設備の標準を整えることに集中する
- **電話回線・番号契約そのものの判断** … 回線の契約・解約・番号の取得は `ベンダー・契約管理` と通信事業者の領域。オフィスITは設定・運用・一次窓口を担い、契約判断を窓口の裁量でしない
- **無秩序な特例配線・特例機材** … 「この部屋だけ特別な機材」「この席だけ手で引いたケーブル」を場当たりで増やさない。標準から外れた1つが、移転・障害時に必ず事故になる

> ここで線を引くことは「縦割りで突き放す」ことではない。**ITが見るべき設備に集中し、総務・ファシリティ・インフラ運用・ベンダーへの引き渡し先を明示する**ためのもの。断るのではなく、正しい行き先へ繋ぐ。

## よくある課題

- **総務との境界が曖昧** … 「電源か機器か」「契約か運用か」で押し付け合いが起き、停電工事・席移動・移転のたびに止まる
- **移転・レイアウト変更で配線が崩壊** … 図面と現物が一致せず、移転後に「どの席のLANが死んでいるか」「どのポートがどこに繋がるか」が分からなくなる
- **会議直前のAVトラブル** … アダプタが無い・映らない・音が出ないで、会議の冒頭が機材トラブルで潰れる。最も発生頻度が高く、最も目立つ
- **予備機・予備部材がない** … 1台壊れたら即業務停止。ケーブル・トナー・電話機・アダプタの在庫がなく、復旧がベンダー手配待ちになる
- **属人的な配線・設定** … 「あの人しか配線が分からない」状態。担当の不在で席移動も障害対応も止まる
- **台帳が現物とずれる** … 設備台帳・配線図が更新されず、現地に行かないと何も分からない

## ベストプラクティス

- **会議室の標準化** … 機材・配線・繋ぎ方を会議室間で統一し、「繋ぐだけで映る」を標準にする。会議室ごとの差をなくすほど、保守も予備も楽になる
- **会議室に現地手順と連絡先を掲示** … 利用者が自力で繋げる手順と、ダメなときの連絡先を会議室に貼る。問い合わせ自体を減らす
- **予備在庫の定数管理** … ケーブル・アダプタ・トナー・電話機・LANケーブルの予備を定数（持つべき数）で管理し、切れる前に補充する。「壊れたら即代替」を在庫で担保する
- **配線図とラベルの徹底** … LANポート・スイッチ・パッチパネルにラベルを貼り、結線情報を図面で維持する。**現物とラベルと図面の三点一致**が、移転・障害時の切り分けを決める
- **移転・レイアウト変更を「プロジェクト」として扱う** … 席替え・増床・移転は片手間でやらない。総務・ファシリティ・ベンダーと事前に役割分担（電源・配線・機器・番号）を決め、図面・台帳の更新までを工程に含める
- **総務との責任分界の文書化（RACI等）** … 「設備ごとに、誰が主担当・誰が承認・誰が実行か」を表で明文化する。停電工事・消耗品・移転の判断を、その都度の押し付け合いにしない
- **ベンダー保守の前提整理** … 複合機（リース）・電話（回線・PBX）はベンダー保守が前提。**一次切り分けはIT、機器修理はベンダー**の線引きと、保守の連絡先・対応時間（SLA：Service Level Agreement、サービス品質の合意）を整理しておく

## 成熟度モデル

自社の現在地を測る目安。

| Lv | 状態 | 特徴 |
|---|---|---|
| Lv1 場当たり | 都度対応 | 会議室・複合機・電話・配線がバラバラ。台帳も配線図もなく、壊れてから慌てる。総務との境界も曖昧 |
| Lv2 把握 | 台帳がある | 設備台帳・配線図があり、何がどこにあるか分かる。総務との責任分界が決まっている |
| Lv3 標準化 | 型で回す | 会議室・機種が標準化され、予備在庫を定数管理。現地手順・掲示で自己解決が進み、移転も役割分担して進められる |
| Lv4 計画運用 | 先回り | 消耗・故障の傾向から更新計画・予備計画を立て、移転・増床を見越して構える。物理が止まる前に手を打つ（プロアクティブ） |

> 段階は目安であり、現実には重なって進む。ただし**まず Lv2（台帳・配線図と総務との境界）を固める**こと。現物がどこにあるか・どこからが誰の仕事かが分からなければ、その先の標準化も移転対応も成り立たない。
