# 社内IT業務 構造化ツリー

社内ITの仕事を **3つの大分類（守り／攻め／基盤・統制）** に整理した分類体系。
人間が自分の現在地を把握するため、またAIが業務を解釈・分類するための共通リファレンスとして使う。

## 分類の基準（守り／攻めの分け方）

**攻め／守りは「目的」で分ける（経済産業省の定義に準拠）。**

- **守りのIT** … コスト削減・既存業務の効率化・安定運用（＝維持＋効率化）
- **攻めのIT** … 売上拡大・新規事業・競争優位など新たな価値の創出（＝成長＋変革）

> 重要：**「内製開発」「データ活用」「自動化」などは“手段”であり、目的で振り分ける。**
> 効率化が目的なら守り、新価値創出が目的なら攻め。同じ技術でも目的次第で所属が変わる（これが経産省の攻め/守りの考え方）。

## このツリーの使い方（現在地の出し方）

自分の業務位置は **「枝 × フェーズ」** の2座標で特定する。

- **枝（What）** … このツリーのどのノードにいるか ＝ 届けたい価値
- **フェーズ（Where）** … その仕事がどの進行段階にあるか

> 例：経理部のExcel集計を自動化中（作り始め）
> → コスト削減が目的なので `守り > 業務効率化・改善 > 手作業の自動化` ／ フェーズ＝**構築・導入**

---

## 1. 守り（守りのIT：維持・運用＋効率化）

経産省定義：コスト削減・既存業務の効率化・安定運用。ITIL／ITSM のプラクティスに対応。

### A. 維持・運用（対応サイクル型）

- **ユーザーサポート（ヘルプデスク）**
  - 問い合わせ・依頼対応（リクエスト管理）
  - 障害・トラブル対応（インシデント管理）
  - 端末管理（入退社・キッティング・周辺機器）
- **アプリケーション・SaaS運用保守**
  - 基幹・業務システムの保守（バージョン更新・障害対応）
  - グループウェア・SaaSの運用（提供・ベンダー連携）
- **インフラ運用**
  - サーバー・クラウド運用
  - ネットワーク運用
  - 監視・バックアップ
- **可用性・事業継続管理**
  - BCP／DR（災害復旧・復旧訓練）
  - 可用性・キャパシティ管理
- **セキュリティ運用**
  - ID・アクセス権限管理（アカウント発行〜棚卸し・権限）
  - 脆弱性・パッチ対応
  - セキュリティインシデント対応
- **運用品質・管理**
  - 問題管理（再発防止）
  - 変更管理
  - IT資産・ライセンス管理
- **オフィスIT・ファシリティ**
  - 会議室AV・複合機・固定／IP電話・配線

### B. 業務効率化・改善（プロジェクト型）

既存業務をITで楽に・安くする（目的＝効率化なので守り）。

- 手作業の自動化（RPA・スクリプト）
- ツール／SaaS導入（効率化目的）
- 業務プロセス改善・標準化
- 効率化のための内製・ローコード開発

**フェーズ軸**：A＝受付 → 対応 → 解決 → 記録（＋PDCA）／ B＝発見 → 分析 → 企画 → 構築 → 定着 → 改善

---

## 2. 攻め（攻めのIT：成長・変革）

経産省定義：売上拡大・新規事業・競争優位・新たな収益モデルの創出。

- **新規事業・新サービスのIT支援**（内製・データ活用を含む、新価値創出が目的のもの）
- **デジタル接点・顧客体験の創出**（EC・Web・新チャネル）
- **データ活用による新価値創出**（売上・意思決定に効く分析・予測、データの新規活用）
- **先進技術による競争力強化**（AI／生成AIで新価値）
- **既存システムの刷新・モダナイゼーション**（レガシー刷新・クラウド移行・システム統合＝「2025年の崖」対応）
- **全社DX・変革の推進**（事業モデル変革のドライブ）

**フェーズ軸（プロジェクト型）**：発見・相談 → 現状分析・要件 → 企画・提案 → 構築・導入 → 定着化 → 効果測定・改善

---

## 3. 基盤・統制（Govern：戦略・ガバナンス）

IT全体の方向づけ・統制を担う上位の仕事。COBIT／TOGAF に対応。

- **IT戦略・企画**
  - 中長期戦略・ロードマップ
  - 予算・コスト管理
  - 投資判断・システム選定
- **プロジェクト・ポートフォリオ管理（PMO）**
  - 案件の優先順位づけ・予算配分
  - 全社横断の進捗・品質管理
- **ガバナンス・統制**
  - ITポリシー・規程
  - 内部統制・監査対応（IT統制・J-SOX）
  - コンプライアンス（個人情報・法令）
  - ITリスク管理（全般）
- **データガバナンス**
  - データ管理方針・品質
  - データ・連携基盤の方針
- **ベンダー・契約管理**
  - ベンダー選定・評価
  - 契約・SLA管理
- **全社アーキテクチャ**
  - システム全体構成・技術標準
- **IT組織・人材マネジメント**
  - 採用・育成・スキル管理
  - 推進体制・組織設計
  - 全社ITリテラシー教育

**フェーズ軸（年次サイクル＋案件サイクル）**：策定 → 運用 → 評価 → 見直し

## 補足：分類が迷いやすい箇所（境界メモ）

- **SaaS／ツール1つでも複数分類にまたがる** … 選定・契約＝統制、導入プロジェクト＝守りB（効率化目的）or 攻め（新価値目的）、稼働後の運用・ライセンス＝守りA。ライフサイクルの段階で置き場が変わる。
- **同じ技術（内製・データ・AI）は目的で振り分け** … 効率化＝守りB／新価値創出＝攻め。
- **アカウント／ID管理は「セキュリティ運用 > ID・アクセス権限管理」に一本化** … 旧版でユーザーサポート側と重複していたのを解消。

## 出典・位置づけ

| 要素 | 根拠 |
|------|------|
| 攻め／守りの区分（目的で分割） | 経済産業省（DXレポート／攻めのIT活用指針 等）。効率化＝守り、成長＝攻め |
| 守り配下の中身 | ITIL／ITSM のプラクティス |
| 基盤・統制配下 | COBIT（統制）／TOGAF（アーキテクチャ） |
| ツリーへの統合・各枝の粒度・フェーズ軸 | 上記の概念を自社向けに再構成した合成（特定の単一出典なし） |

個々の用語・大分類は既存フレームワークに準拠。ツリーの構造自体は自社向けのカスタム整理。
